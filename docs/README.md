# Documentation

This folder holds SDLC artefacts for the project.

Artefacts should be created in sequence:

1. `brd/` for BRD-derived notes and traceability.
2. `agile/` for epics, user stories, acceptance criteria, and backlog.
3. `testing/` for test strategy, test cases, and traceability matrix.
4. `design/` for architecture, API, data, agent workflow, security, and deployment design.
5. `devsecops/` for CI/CD, build, deployment, and security automation design.
6. `operations/` for runbooks, release notes, monitoring, and support notes.

Do not create detailed SDLC deliverables until the user asks for the corresponding stage.
