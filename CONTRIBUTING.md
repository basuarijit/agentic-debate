# Contributing

## Development Flow

1. Work from the BRD and approved SDLC artefacts.
2. Keep backend work inside `backend/`.
3. Keep frontend work inside `frontend/`.
4. Add or update tests with implementation changes.
5. Keep secrets out of source control.
6. Update relevant documentation when behavior, architecture, or deployment changes.

## Branching

Use feature branches for implementation work once development begins.

Suggested branch naming:

- `feature/<short-description>`
- `fix/<short-description>`
- `docs/<short-description>`
- `devsecops/<short-description>`

## Pull Requests

Pull requests should describe:

- Requirement or user story reference.
- What changed.
- Tests performed.
- Security or deployment impact.
